Polish it up

We've got a pretty basic application. We only go through the list once, and if all of the tasks aren't finished, then they are abandoned and we never check again to see if they are done. That's not very helpful! We can do better than that.

Let's add a new costume for our sprite that looks like a button. Then the user can click the sprite/button to go through the list again and remove any newly completed items.

We can drag-and-drop an image onto the costume tab to create a new costume from that image. Right-click on the image below to save it to your computer, then use it to make a button costume for your sprite.


Use the when I am [clicked] block to broadcast a "completed task" message. When that message is received, then use your custom block to remove completed tasks.


Rise to the Challenge

Show the List

We can display the To Do list by showing the global list variable but that isn't very pretty. Try to make additional sprites to show each item from the to do list on the stage. By using a say block without a time, the say will remain visible.

This approach will limit the number of items that you can put into the list and still have it display on the stage. Figure out what amount of tasks will show nicely and add code to ensure that there are never more tasks added than can be displayed.

Add More Tasks

Provide a way for the user to add more tasks to the list without starting over again. You could make another "button" or use a hot-key (keypress event).
