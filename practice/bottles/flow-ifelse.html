Review: Conditional Program Flow

Generally speaking, programs run sequentially top to bottom, but some blocks can alter that flow before allowing the script to continue on to the next block. We've seen how a repeat block can alter that sequential flow. Using a custom block also alters that flow, by causing the steps inside that block to run before the main script goes on to the next block.

Another way to alter the flow is with conditional logic. For this exercise we will look more closely at if and if-else blocks.

Simple if

First let's look at the if block. This will cause something to happen when some condition (a boolean expression) is true. If that condition is not true, then nothing happens. The program just continues with the next block in the sequence.

Here is the example that we saw in lab for a leaving the house routine:
- grab the keys
- put on shoes
- if it's raining, (then) grab a jacket
- go out the door

The most imporant thing to notice in this example is that you always will grab the keys, put on shoes, and go out the door. However, it is only sometimes - when it's raining - that you will you grab a jacket. The if statement allows us to do something (like grab a jacket) conditionally. If the condition is not true (if it is false), then we skip it and move on to the next step.

Remember that in programming, yes and no is represented by the boolean values for true and false.

How do you decide if you want an if block or an if-else block?

Remember that both will do something when something is true. If you need to do something else when the condition is false, then you need the if-else block. If the rest of what happens is the same in both cases, then use the if block.
