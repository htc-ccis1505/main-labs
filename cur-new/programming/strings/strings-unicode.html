Unicode

Deep down, computers deal with numbers, specifically 1's and 0's or binary numbers. Unicode is a way to represent characters a number. It is one of many different ways to encode characters, but is commonly used now because it is able to consistently encode every character, no matter what language the character is in, what programming language you use, or what type of computer you are working on.

There are two additional blocks in Snap that allow us to convert between characters and their unicode value:

The block will tell you the unicode, numeric value of a character. This block will report that the unicode of (a) is 97.

The block will tell you what character (or letter), is represented by a numeric unicode value. This block will report that unicode (65) as a letter is A.

Notice that while we think of both "a" and "A" as the letter a they are completely different characters, each with their own unicode representation.

The unicode value of a character is not something that we need often, but it can come in handy for certain types of programming problems. In particular it is handy to turn strings into numbers for things like encryption and checking for data loss or data tampering. The process of doing this is often called hashing. While these problems can be quite complex, they are built on this fundamental idea of converting data into a numeric representation.
