Objects

In programming, an object is a way to group related data properties and functionality into one package. It is intended to be a very intuitive way to organize your code. Some of the typical examples for illustrating objects and how they work show these concepts well - shapes, animals and people. We'll look at a few examples throughout this lab.

When designing an object - thinking about what we will code before we start doing it, we think about it's properties and behaviors. The object properties are things that describe the object. These are nouns, like a person's name, age, or height. The object behaviors are verbs, things that can be done to the object, like change a name, or that the object itself can do, like grow taller or get older.

Make a Person Object

To design a person object, we start by listing the properties and behaviors we care about for our program. There are many things that a person can do and many things that describe them, but we only care about ones that we need for our application. My application will track names and ages only. We can add people, and they can say their name and age. Simple enough, right?

So what will my object look like?
Properties:
- name
- age

Behaviors:
- say_name_and_age

Notice I left out the height property I mentioned earlier as it isn't relevant to my application. While I could add a height property, since it isn't used that makes thing unnecessarily complex. I will Keep It Simple by not adding it. It might be important in another application, but not in this one. That means a Person object might look and act differently in other applications.

Let's look at how to write this in python: 


class Person:
    name = ''
    age = 0

    def say_name_and_age(self):
        print("Hello, my name is " + self.name + " and I'm " + str(self.age) + " years old.")

Notice that we start by creating a class. A class is a template for creating objects. The class is the pattern or blueprint and it can create multiple individual objects. The properties are just variables and the behaviors are just functions - though we call functions in a class methods.

Notice that in the say_name_and_age method we include a parameter called self. This is important as it provides the access to the name and age properties - self.name and self.age. Just using name or age would result in an error.

To create a Person object and have it say the name and age: 


person1 = Person()
person1.say_name_and_age()

This person doesn't have a name and is 0 years old. Do you see why? When we wrote the class above, we set the name to the empty string '' and set the age to 0.

It's not very helpful if all our people are nameless, but we can set the name and the age: 


person1.name = "Buffy"
person1.age = 16
person1.say_name_and_age()

We can make multiple people following the same pattern: 


person2 = Person()
person2.name = "Willow"
person2.age = 15
person2.say_name_and_age()

Notice that we use the variable name followed by a dot to access the properties and behaviors of that specific object. We've been doing this in the previous labs as we've worked with strings, lists, and dictionaries. What does that tell you?

Python strings, lists and dictionaries are all objects that are built-in to the language.
