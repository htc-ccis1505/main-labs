Encapsulation/h2>

There are three key principles of Object Oriented Programming:
- Encapsulation
- Inheritance
- Polymorphism We'll look at each of these in the remainder of this lab.

Let's add a new method say_hello that doesn't use those properties: 


class Person:
    name = ''
    age = 0

    def say_name_and_age(self):
        print("Hello, my name is " + self.name + " and I'm " + str(self.age) + " years old.")

    def say_hello():
        print("Hello!")

If we run this now by saying person1.say_hello(), we get the following error: 


Traceback (most recent call last):
  File "person.py", line 31, in 
    person1.say_hello()
TypeError: say_hello() takes no arguments (1 given)

Even if we don't need the object properties, you must add the self parameter to all object methods.

Fix the code by adding the self parameter: 


class Person:
    name = ''
    age = 0

    def say_name_and_age(self):
        print("Hello, my name is " + self.name + " and I'm " + str(self.age) + " years old.")

    def say_hello(self):
        print("Hello!")



Another common error is to forget to use the self parameter inside the method. Modify the say_name_and_age method in the class by removing the self from the print statement: 


class Person:
    name = ''
    age = 0

    def say_name_and_age(self):
        print("Hello, my name is " + name + " and I'm " + str(age) + " years old.")

    def say_hello(self):
        print("Hello!")

If we run this now by saying person1.say_name_and_age(), we get the following error: 


Traceback (most recent call last):
  File "person.py", line 11, in 
    person1.say_name_and_age()
  File "person.py", line 6, in say_name_and_age
    print("Hello, my name is " + name + " and I'm " + str(age) + " years old.")
NameError: global name 'name' is not defined

If you forget to use the self parameter to refer to a property or method, you will get an error.

It is important that you learn to recognize these error messages and know how to fix them.
